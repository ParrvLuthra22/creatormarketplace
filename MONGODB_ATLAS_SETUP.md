# MongoDB Atlas Setup Guide

## Step 1: Create MongoDB Atlas Account

1. Go to https://www.mongodb.com/cloud/atlas/register
2. Sign up with your email or Google account
3. Verify your email address

## Step 2: Create a Free Cluster

1. Click **"Build a Database"**
2. Select **"M0 FREE"** tier
3. Choose your cloud provider:
   - **AWS** (recommended)
   - Google Cloud
   - Azure
4. Select region closest to your deployment:
   - For US deployments: `us-east-1` or `us-west-2`
   - For Europe: `eu-west-1` (Ireland)
   - For Asia: `ap-south-1` (Mumbai)
5. Cluster Name: `CreatorMarketplace` (or keep default)
6. Click **"Create"** (takes 3-5 minutes)

## Step 3: Configure Database Access

1. In left sidebar, click **"Database Access"**
2. Click **"Add New Database User"**
3. Authentication Method: **Password**
4. Username: `creatormarketplace`
5. Password: Click **"Autogenerate Secure Password"** and **SAVE IT**
6. Database User Privileges: **"Read and write to any database"**
7. Click **"Add User"**

## Step 4: Configure Network Access

1. In left sidebar, click **"Network Access"**
2. Click **"Add IP Address"**
3. For development/testing:
   - Click **"Allow Access from Anywhere"**
   - IP Address: `0.0.0.0/0`
   - Click **"Confirm"**
4. For production (recommended):
   - Add your deployment platform's IP addresses
   - Vercel: `0.0.0.0/0` (Vercel uses dynamic IPs)
   - Railway: Get IPs from Railway dashboard
   - Render: Get IPs from Render dashboard

## Step 5: Get Connection String

1. Go to **"Database"** in left sidebar
2. Click **"Connect"** on your cluster
3. Choose **"Connect your application"**
4. Driver: **Node.js**
5. Version: **5.5 or later**
6. Copy the connection string (looks like):
   ```
   mongodb+srv://creatormarketplace:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```
7. Replace `<password>` with the password you saved in Step 3
8. Add database name before the `?`:
   ```
   mongodb+srv://creatormarketplace:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/creator-marketplace?retryWrites=true&w=majority
   ```

## Step 6: Update Environment Variables

### For Local Testing
Update `backend/.env`:
```env
DATABASE_URL=mongodb+srv://creatormarketplace:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/creator-marketplace?retryWrites=true&w=majority
```

### For Production Deployment

#### Vercel
1. Go to your Vercel project
2. Settings → Environment Variables
3. Add:
   - `DATABASE_URL` = `mongodb+srv://...`
   - `JWT_SECRET` = (generate strong secret)
   - `NODE_ENV` = `production`
   - `CORS_ORIGIN` = `https://yourdomain.com`

#### Railway
1. Go to your Railway project
2. Variables tab
3. Add same variables as above

#### Render
1. Go to your Render service
2. Environment tab
3. Add same variables as above

## Step 7: Test Connection

Run your backend locally with the new connection string:
```bash
cd backend
npm run dev
```

You should see:
```
✅ MongoDB connected successfully
🚀 Server running on port 5001
```

## Troubleshooting

### Error: "Authentication failed"
- Double-check username and password
- Make sure password doesn't contain special characters that need URL encoding
- Use `encodeURIComponent()` for special characters

### Error: "IP not whitelisted"
- Add `0.0.0.0/0` to Network Access
- Wait 1-2 minutes for changes to propagate

### Error: "Connection timeout"
- Check your internet connection
- Verify the connection string is correct
- Ensure cluster is running (not paused)

## Security Best Practices

1. **Never commit `.env` files** - Already in `.gitignore`
2. **Use strong passwords** - Autogenerated passwords are best
3. **Rotate secrets regularly** - Change JWT_SECRET periodically
4. **Whitelist specific IPs** - In production, avoid `0.0.0.0/0` if possible
5. **Enable audit logs** - Available in paid tiers
6. **Set up alerts** - Monitor connection spikes
